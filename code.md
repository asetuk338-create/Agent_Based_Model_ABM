# AI 互动生存博弈：基于 Q-Learning 的差异化价格竞争模拟

## 项目概述
本项目模拟了一个双寡头垄断市场中的差异化价格竞争环境。两家AI智能体通过Q-Learning算法学习定价策略，在追求利润最大化的过程中，可能形成合谋、竞争或价格战等不同市场格局。项目完整展示了从市场环境构建到AI学习决策的全过程。

---

## 一、市场环境的构建 (class model)
*这是AI互动生存的“物理法则”，定义了它们能定什么价格，以及定了该价格能赚多少钱。*

### `demand(self, p)`
**需求计算器**
- 基于经济学中的Logit需求模型构建
- 输入参数：`p` - 两家AI给出的价格向量 [p1, p2]
- 计算逻辑：考虑产品差异化程度和消费者效用函数
- 输出结果：两家企业各自能瓜分到的市场份额 [share1, share2]
- 经济学原理：价格越高，市场份额越小；差异化越大，价格敏感度越低

### `compute_profits(self, p)`
**利润结算中心**
- 计算公式：(价格 - 边际成本) × 市场需求
- 输入参数：`p` - 当期价格组合 [p1, p2]
- 计算过程：先调用demand()获取市场份额，再乘以单位利润
- 输出结果：两家企业在当期价格组合下获得的真实利润 [profit1, profit2]
- 注意：这里的利润就是强化学习中的即时奖励(Reward)

### `init_actions(self)`
**离散动作空间构建**
- 通过求导计算理论上的价格边界：
  - **竞争底价（Nash）**：古诺-纳什均衡价格
  - **垄断高价（Monopoly）**：联合利润最大化价格
- 在这个价格区间内均匀切分出k个价格档位（默认k=15）
- 这15个离散价格构成AI每天只能勾选的“定价菜单”
- 意义：简化学习空间，同时保留经济学的理论边界

### `init_Q(self)`
**生成AI的“大脑”**
- 为每个AI初始化一个Q矩阵
- 矩阵维度：状态数 × 动作数
  - 状态数 = 记忆长度k（对手上一期的价格）
  - 动作数 = 可选价格数k
- 初始策略假设：AI会盲目假设对手是完全随机定价的
- Q值初始化：通常设为0或根据先验知识设置

## 二、AI的决策与学习机制 (Q-Learning Functions)
*这部分代码赋予了AI学习和博弈的能力。*

### `pick_strategies(game, s, t)`
**ε-贪心决策（探索 vs 利用）**

核心参数：
- 探索概率 ε = e^(-βt)，随时间指数衰减
- β：探索衰减系数，控制学习速度

决策流程：
1. 生成随机数与ε进行比较
2. **探索期（Exploration）**：随机数 < ε
   - AI闭着眼睛在15个价格里随便挑一个
   - 虽然可能导致短期亏损，但能帮AI探索未知的高利润区域
   - 避免陷入局部最优

3. **利用期（Exploitation）**：随机数 ≥ ε
   - AI查阅自己的Q矩阵
   - 在当前状态s（对手上一期定的价格）下
   - 选择预期总收益最高的价格

### `update_q(game, s, a, s1, pi, stable)`
**经验更新（核心大脑）**

这是整个代码的灵魂，使用贝尔曼方程(Bellman Equation)更新Q值：

$$Q(s,a) \leftarrow (1-\alpha)Q(s,a) + \alpha(r + \delta \max_{a'} Q(s',a'))$$

参数解释：
- α：学习率，控制新信息覆盖旧信息的速度
- δ：折扣因子，AI对未来的重视程度
- r：当期获得的利润(pi)
- s：当前状态（本期对手价格）
- a：采取的动作（本期自己的价格）
- s1：下一状态（下期对手价格）

**博弈体现：**
- **即时满足**：今天赚了多少（r）
- **长远眼光**：预判对手反应，估算明天能赚多少（δ·maxQ）
- **战略思维**：正是因为关注未来，AI才能学会：
  - “如果我今天恶意降价抢市场，明天必然遭遇对手的价格战报复”
  - 从而克制降价冲动，维持高价合谋

### `simulate_game_with_history(game)`
**主循环与监控**

功能职责：
1. 驱动时间齿轮不断向前转动
2. 状态管理：将上一期的结果转为下一期的初始状态
3. 循环调用：反复执行 决策 → 交互 → 学习 的循环
4. 数据记录：记录历史价格序列，供后续画图分析
5. 终止条件判断：
   - **稳定收敛(tstable)**：价格波动小于阈值，达成均衡
   - **强行终止(tmax)**：防止无限循环，设置最大迭代次数

---

## 三、关键参数说明

| 参数 | 符号 | 作用 |
|------|------|------|
| 价格档位数 | k=15 | 离散动作空间大小 |
| 探索衰减系数 | β | 控制探索率衰减速度，β 越大，epsilon降到接近 0 的速度就越快 |
| 学习率 | α | Q值更新步长，α接近0则更念旧。原文α=0.15，如果α太高（非常非常重视现在的信息，AI 只要一经历价格战，就会忘掉合谋带来的好处，导致合谋极度脆弱） |
| 折扣因子 | δ |  乘在未来预期利润前的一个系数（取值 0 到 1），论文中设定为0.95，代表AI对未来的重视程度，正是因为AI重视未来收益才会选择合谋 |
| 最大迭代数 | tmax | 防止无限循环 |
| 稳定阈值 | stable | 判定收敛的波动范围 |

---

## 四、运行流程

1. 初始化市场环境(model)
2. 初始化两家AI的Q矩阵
3. 进入主循环：
   - 根据当前状态选择价格
   - 计算市场需求和利润
   - 更新Q值
   - 记录历史数据
   - 检查终止条件
4. 输出分析结果和可视化图表
